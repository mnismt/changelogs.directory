# Logo Generation Prompt for Changelogs.directory

## Overview

This document contains the prompt for generating an abstract, monochrome logo for the Changelogs.directory platform.

## Prompt

Create an abstract, minimalist logo for "Changelogs.directory" - a platform that tracks changelog information for CLI developer tools.

### Design Requirements

- **Strictly monochrome**: pure black and white only, NO colors, NO gradients
- **Abstract and geometric** shape that represents change, evolution, and version tracking
- **Developer/terminal aesthetic**: clean, technical, code-inspired
- **Minimal and modern** with high contrast
- Should work well at **small sizes** (16x16px) and large formats
- Suitable for **dark backgrounds** (#0A0A0A black)

### Conceptual Elements to Convey

- **Change and evolution**: progressive layers, transformation, or flow
- **Version history**: stacked elements, timeline, or iteration
- **Directory/organization**: structured, ordered, cataloged
- **CLI/terminal aesthetic**: geometric, sharp edges, monospace-inspired

### Style References

- Think GitHub's Octocat simplicity but more abstract
- Terminal window aesthetics
- Version control branch diagrams
- Diff markers (+/- symbols) abstracted
- Layered pages or stacked documents concept

### Output Format

- Simple geometric shapes
- High contrast black on white (or white on black)
- No text, just the symbol/mark
- Clean lines, sharp edges
- Scalable vector-friendly design
- Square or slightly rectangular aspect ratio

### Avoid

- ❌ Any colors beyond pure black/white
- ❌ Gradients or shadows
- ❌ Overly detailed or illustrative elements
- ❌ Literal representations (no actual documents, computers, etc.)
- ❌ Busy or complex patterns
